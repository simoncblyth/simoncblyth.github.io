Latex Tables Compilation¶

compilation-update
invokes the below -fix and -make
compilation-fix
inplace editing of tables to fix latex errors
compilation-fixkey
inplace editing of key
compilation-make
recursive flattening many documents into one, latexing, creating PDF and tar.gz of the tex invokes ~/h/compilation/Makefile from compilation-dir
compilation-pagination
use while fixing the pagination of tables : curl GETs 00000-table_body.tex which defines the structure (newpage cuts and shrinkages) based on the ICPAGS and IPOLES in the enironment (remember to update the environment with scrape- after edits)
compilation-asciicheck
check for non-ascii chars in the compilation latex document
compilation-open
open the compilation PDF
compilation-sync
synonym for compilation-scp-to-downloads which scps the pdf to H:$(compilation-server-htdocs)/downloads/
compilation-tgztest
downloads the tarball from http://$HEPREZ_SERVER/downloads/$B2C_COMPILATION_NAME.tar.gz and compiles the latex into a PDF. Also invokes compilation-asciicheck
compilation-grab
download the compilation tarballs, unpack them and compare the table captions

HOW TO PROPAGATE SVN BLURB UPDATES INTO COMPILATION AND SYNC TO DOWNLOADS¶

Iterate on 1 and 2 until good enough:

  1. vi ~/heprez/compilation/blurb.tex
  2. compilation- ; compilation-update

NB slightly tricksy compilation-make operation of using distant Makefile from the 00000 dir that contains the tables:

make -f $HEPREZ_HOME/compilation/Makefile

This copies the SVN blurb.tex into the versioned one with name like b2charm_end_of_2011_v005_blurb.tex

It really is just a copy:

simon:compilation blyth$ diff $(compilation-fold)/b2charm_end_of_2011_v005_blurb.tex ~/heprez/compilation/blurb.tex

Once converged:

  1. svn ci ~/heprez/compilation/blurb.tex -m ”... “

  2. compilation- ; compilation-update

  3. sync to downloads:

    compilation-sync  # default is HEPREZ_SERVER namely cms01.phys.ntu.edu.tw
compilation-sync  hfag.phys.ntu.edu.tw

  4. notify CVS updater

TO FIX PAGINATION AND SHRINKAGES¶

  1. identify overflowed tables and set the inputs to compilation/shrink.py run it to create the new poles
  2. create structure + latex doc + view pdf compilation-pagination;compilation-update;compilation-open
  3. update HFAG_IPOLES envvar with scrape-vi to mark tables that start new pages and ones that need a small font (via negation)
